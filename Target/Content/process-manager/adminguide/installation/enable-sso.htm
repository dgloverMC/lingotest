Enabling single sign-on via SAML (optional)

You can enable single sign-on (SSO) via Security Assertion Markup Language (SAML) for your on-premises installation. Read more about this topic in the Signavio Process Manager User Guide. The user guide article also explains, which steps to take on the side of your service provider (SP), Signavio Process Manager.

On-premises customers have two options to configure SSO for an identity provider (IdP) which differ from the description for SaaS workspaces.

  1. First option: To configure SSO for your IdP, follow your IdpÂ´s specific configuration. You have to set Signavio Process Manager as a trusted party.
  2. Second option: If XML metadata is required to configure SSO, open the following XML metadata template:

    XML metadata template

    You need to replace the following placeholders in the template with your specific data:
    1. ###SERVER_PARAMETER_FROM_CONFIGURATION_XML_WITHOUT_PROTOCOL###: Use the URL of your Signavio Process Manager system. You specified the URL in your configuration.xml when you configured Signavio Process Manager. An example is example.com.

    2. ###SERVER_PARAMETER_FROM_CONFIGURATION_XML_WITH_PROTOCOL###: Use the URL, including the protocol, of your Signavio Process Manager system. You specified the URL and the protocol in your configuration.xml when you configured Signavio Process Manager. An example is https://example.com.

Ensure to set the following SAML response attributes in your IdP configuration:

  • email
  • first_name
  • last_name
  • Name ID
